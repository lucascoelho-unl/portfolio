# portfolio
To pull off a seamless Generative UI where an agent streams beautiful, interactive cards directly into a chat, you need a stack that is heavily optimized for Server-Sent Events (SSE) and component streaming.

Here is the exact production-grade stack to build this:

### 1. The Core Framework: Next.js (App Router)

For Generative UI, Next.js is currently the absolute best tool for the job. You need a framework that deeply supports React Server Components (RSC).

* **Why:** Instead of sending raw JSON from the server to the client and having the client figure out how to render it, Next.js allows your server to generate the React component itself and stream it directly to the browser.

### 2. The Generative UI Bridge: Vercel AI SDK

This is the secret sauce. Writing the glue code between an LLM's text stream and a React component is incredibly tedious. The Vercel AI SDK (`ai/rsc`) is purpose-built for this exact architecture.

* **Why:** It has a specific function called `streamUI`. When your agent decides to call a tool (e.g., `getDesignStudioProjects`), the SDK intercepts that tool call and allows you to yield a `<ProjectCard/>` component directly into the chat feed while the LLM continues generating text.

### 3. The Agentic Backend: LangGraph or MastraAI

You need an orchestrator to handle the actual "thinking," context management, and routing before the UI is ever rendered.

* **Why:** For complex multi-agent architectures or autonomous workflows, LangGraph excels at defining state machines and looping behaviors. If you want a slightly more lightweight, TypeScript-native approach specifically for managing tool execution and LLM routing, MastraAI is exceptionally fast. Both integrate cleanly with Next.js API routes.

### 4. The Visuals & Styling: Tailwind CSS + Framer Motion + shadcn/ui

Since the goal is a "very pretty and fancy UI," you want a system that looks high-end out of the box but is deeply customizable.

* **shadcn/ui:** Gives you beautifully designed, accessible base components (buttons, input fields, cards) that you fully control.
* **Framer Motion:** Crucial for the chat feel. When a new card is generated by the agent, it shouldn't just instantly snap into existence. Framer Motion lets you easily add a smooth `y: 20, opacity: 0` to `y: 0, opacity: 1` spring animation so the UI components organically slide into the chat feed.

### 5. Native Internationalization (Next-intl)

To make the portfolio globally accessible, building in native support for English, Portuguese, and Spanish from day one is a huge technical flex.

* **Why:** `next-intl` integrates perfectly with the Next.js App router. You can feed the active locale state directly into your agent's system prompt, ensuring the AI not only responds in the correct language but also fetches the appropriately localized UI cards.

### The Workflow in Action:

1. **User Types:** "Show me what you built for your Design Studio pitch."
2. **Next.js Server:** Receives the prompt and passes it to your **LangGraph/MastraAI** agent.
3. **The Agent:** Recognizes the intent, triggers the `fetchDesignStudioData` tool, and generates a structured response.
4. **Vercel AI SDK:** Takes that tool call and yields a React `<StartupPitchCard/>`.
5. **The Client:** Receives the component and uses **Framer Motion** to smoothly animate the beautiful UI card into the chat window.
